문자열은 파이썬에서 가장 많이 다루는 자료형입니다. 인덱싱과 슬라이싱 개념을 "기차" 비유로 이해시킵니다.

오늘 배울 것

  • 인덱싱으로 글자 하나 꺼내기
  • 슬라이싱으로 부분 문자열 자르기
  • 유용한 문자열 메서드 사용법
인덱싱, 슬라이싱, 메서드 세 가지를 배웁니다.

문자열이란?

🚂 문자열은 글자들이 줄 선 기차야!
각 칸에 글자가 하나씩 타고 있어

P y t h o n
0 1 2 3 4 5
기차 비유와 함께 인덱스 번호를 보여주세요. 번호는 0부터 시작한다는 것이 핵심입니다!

인덱싱: 글자 하나 꺼내기


word = "Python"

print(word[0])    # P  (첫 번째)
print(word[1])    # y  (두 번째)
print(word[5])    # n  (여섯 번째)
인덱스는 0부터 시작해!
첫 번째 글자 = [0], 두 번째 = [1], ...
0-based indexing은 프로그래밍의 기본입니다. "기차 칸 번호가 0번부터 시작한다"고 설명하세요.

뒤에서부터: 음수 인덱스


word = "Python"

print(word[-1])    # n  (마지막)
print(word[-2])    # o  (뒤에서 두 번째)
print(word[-6])    # P  (뒤에서 여섯 번째 = 첫 번째)
0 1 2 3 4 5
P y t h o n
-6 -5 -4 -3 -2 -1
음수 인덱스는 뒤에서부터 세는 방법입니다. -1은 마지막, -2는 마지막에서 두 번째입니다.

len(): 길이 재기


word = "Python"
print(len(word))      # 6

name = "안녕하세요"
print(len(name))      # 5

empty = ""
print(len(empty))     # 0
len() = length(길이)의 약자
마지막 인덱스는 항상 len() - 1이야!
len()은 문자열의 길이를 반환합니다. "Python"은 6글자이지만, 마지막 인덱스는 5입니다 (0부터 시작하므로).

슬라이싱: 부분 자르기


word = "Python"

print(word[0:3])    # Pyt  (0번~2번)
print(word[2:5])    # tho  (2번~4번)
print(word[0:6])    # Python (전체)
[시작:끝] → 시작부터 끝-1까지!
끝 번호는 포함 안 돼!
[0:3]은 0, 1, 2번만 (3번은 빠져!)
슬라이싱에서 끝 인덱스가 포함되지 않는 것이 가장 헷갈리는 부분입니다. "끝 번호 직전까지"라고 강조하세요.

슬라이싱 생략 문법


word = "Python"

print(word[:3])     # Pyt  (처음~2번)
print(word[3:])     # hon  (3번~끝)
print(word[:])      # Python (전체 복사)
  • [:3] = 처음부터 3번 직전까지
  • [3:] = 3번부터 끝까지
  • [:] = 전체
생략 문법은 매우 자주 사용됩니다. 앞을 생략하면 0부터, 뒤를 생략하면 끝까지입니다.

슬라이싱 스텝


word = "Python"

print(word[::2])     # Pto  (2칸씩 건너뛰기)
print(word[1::2])    # yhn  (1번부터 2칸씩)

뒤집기 마법! [::-1]


word = "Python"
print(word[::-1])    # nohtyP (뒤집기!)

name = "토마토"
print(name[::-1])    # 토마토 (회문!)
[::-1]은 문자열 뒤집기의 정석!
회문(앞뒤가 같은 말) 판별에 쓸 수 있어
[::-1]은 매우 유용한 패턴입니다. 회문(palindrome) 판별: word == word[::-1]

문자열 메서드


text = "Hello Python"

print(text.upper())     # HELLO PYTHON
print(text.lower())     # hello python
print(text.title())     # Hello Python

# 원본은 변하지 않아!
print(text)             # Hello Python
문자열은 불변(immutable)!
메서드는 새 문자열을 반환하고, 원본은 그대로야
문자열의 불변성은 중요한 개념입니다. 메서드를 호출해도 원본은 변하지 않습니다. 결과를 사용하려면 새 변수에 저장해야 합니다.

문자열 메서드: 찾기 / 바꾸기


text = "나는 파이썬을 좋아해"

# 찾기
print(text.find("파이썬"))     # 3  (위치)
print(text.count("나"))        # 1  (개수)
print("파이썬" in text)        # True

# 바꾸기
new = text.replace("파이썬", "코딩")
print(new)    # 나는 코딩을 좋아해
find()는 위치를 반환하고, 못 찾으면 -1을 반환합니다. replace()는 새 문자열을 반환합니다 (원본 변경 없음).

split()과 join()


# split: 문자열 → 리스트
sentence = "나는 코딩을 좋아해"
words = sentence.split()
print(words)    # ['나는', '코딩을', '좋아해']

# join: 리스트 → 문자열
result = "-".join(words)
print(result)   # 나는-코딩을-좋아해
split()은 쪼개기, join()은 합치기!
World 3의 input().split()이 이거였어!
split()은 World 3에서 이미 사용해본 적 있습니다. 그때는 이해 없이 사용했지만, 이제 원리를 알게 됩니다.

자주 하는 실수

인덱스 범위 초과! 
word = "abc"      # 길이 3, 인덱스 0~2
print(word[3])     # IndexError!
IndexError: string index out of range
마지막 인덱스 = len() - 1 
word = "abc"
print(word[2])     # c  (마지막)
print(word[-1])    # c  (더 안전!)
IndexError는 존재하지 않는 인덱스에 접근할 때 발생합니다. 음수 인덱스 -1을 사용하면 마지막 글자에 안전하게 접근할 수 있습니다.

자주 하는 실수

슬라이싱 끝 포함 착각! 
word = "Python"
print(word[0:2])   # Py (P, y만! t는 빠져!)

"Pyt" 나올 줄 알았는데 "Py"만 나와!

[시작:끝] → 끝은 미포함! 
print(word[0:3])   # Pyt (0,1,2번 = 3글자)
공식: 글자 수 = 끝 - 시작
[0:3] → 3 - 0 = 3글자!
"끝 인덱스 미포함"은 슬라이싱에서 가장 흔한 혼동입니다. 끝 - 시작 = 글자 수라는 공식을 알려주세요.

오늘 배운 것 정리

  • [0]으로 인덱싱, [-1]로 마지막 글자
  • [시작:끝] 슬라이싱, 끝은 미포함!
  • [::-1]로 문자열 뒤집기
  • .upper() .lower() .replace() .split()
  • 문자열은 불변! 메서드는 새 문자열을 반환
핵심 정리를 확인합니다.

문자열 메서드 요약

메서드기능예시
.upper()대문자로"abc" → "ABC"
.lower()소문자로"ABC" → "abc"
.strip()양쪽 공백 제거" hi " → "hi"
.replace(a,b)a를 b로 교체"cat" → "bat"
.find(x)x의 위치"hello".find("l") → 2
.count(x)x의 개수"hello".count("l") → 2
.split()나누기"a b" → ["a","b"]
